# 共享策划输入验收记录

- 日期：2026-07-22
- 范围：Amazon Listing / A+ 与淘宝 / 天猫准备页、共享输入评估、策划编排、session/run 快照和旧数据兼容
- 决策：复用现有多模态策划模型，不增加独立图片分析页面或第二次 AI 调用

## 验收结论

本轮实现通过。两平台均可从资料库、手动资料或纯商品图开始；无商品档案时在提交阶段自动创建本地草稿。主按钮统一为“生成图片策划”，输入质量统一标记为 `standard`、`image-only`、`facts-only` 或 `empty`，只有完全空输入禁止提交。

策划模型和输入签名只接收本次勾选的商品参考图。未勾选素材和风格参考图不影响完整度，也不会导致策划过期。纯图片任务在模型不支持读图时明确阻止；支持读图时只能把可见内容作为图片证据，不能猜测尺寸、材质、功效或认证。

`PlatformSession` 和 `ProductionRun.contextSnapshot` 已保存可选 `planningInput`：来源、质量、缺项、文字输入、勾选参考图和资料库来源版本。旧记录缺少该字段时继续读取，不执行破坏性迁移。策划失败时保留草稿商品、文字和已上传图片。

## 人工可视检查

- `1440x900`：Amazon 与淘宝主按钮位置和尺寸一致，质量提示稳定，来源切换无重叠；Amazon 非必要参数默认折叠。
- `390x844`：沿用现有小于 900px 的桌面工作台门禁；提示框完整，无横向溢出、裁切或遮挡。
- 应用内浏览器控制台未发现页面运行错误。

## 剩余风险

- 生产构建仍有现有的单 chunk 超过 500 kB 提示，不阻断交付。
- 浏览器自动化仍记录一个已知的非阻断 404 资源备注。
- 真实外部 Provider 的在线可用性、CORS、配额和多模态输出质量不属于本地确定性验证范围。
